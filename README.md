# modrinth_server

A bash script that downloads a Modrinth modpack and sets up a fully functional Minecraft Forge server.

---

## Usage

```bash
./setup.sh <slug> [options]
```

### Arguments

| Argument | Description |
|---|---|
| `<slug>` | **Required.** The Modrinth project slug (e.g. `yet-another-pack`). |
| `--version <version>` | Pack version to install. Omit or leave empty to use the latest release. |
| `--no-auto-update` | Disable automatic update checks on server launch. Default: auto-update **enabled**. |
| `--agree-eula` | Automatically accept the [Minecraft EULA](https://aka.ms/MinecraftEULA). You must agree to it to run a server. |

### Examples

```bash
# Install latest version, auto-update enabled
./setup.sh yet-another-pack

# Install a specific version, no auto-update
./setup.sh yet-another-pack --version 1.0.1 --no-auto-update

# Install latest, accept EULA automatically
./setup.sh yet-another-pack --agree-eula
```

## What the script does

### `setup.sh` — Initial setup

1. **Parses arguments** — slug, version, auto-update preference, EULA acceptance.
2. **Fetches pack metadata** from the Modrinth API to resolve the requested version (or latest).
3. **Downloads the `.mrpack` file** for the resolved version.
4. **Reads pack metadata** from `modrinth.index.json` inside the `.mrpack` to determine:
   - Minecraft version (e.g. `1.20.1`)
   - Mod loader and version (e.g. Forge `47.4.16`)
5. **Creates the `server/` directory.**
6. **Downloads and runs the Forge installer** (`--installServer`) to install Forge and the Minecraft server jar into `server/`.
7. **Extracts config overrides** from the `.mrpack`'s `overrides/` folder into `server/`.
8. **Downloads all server-side mods** listed in `modrinth.index.json`, skipping any with `"server": "unsupported"`.
9. **Removes known client-only mods** that would crash the server (e.g. Connector-wrapped Fabric mods with `environment: client`).
10. **Fixes file permissions** — files extracted from zip archives are often read-only, which causes `AccessDeniedException` on first run.
11. **Writes `server/eula.txt`** if `--agree-eula` was passed.
12. **Writes a `server/.setup-config.json`** recording the slug, installed version, and auto-update preference.
13. **Generates `server/run.sh`** — a launch script that:
    - Reads `.setup-config.json` to check if auto-update is enabled.
    - If enabled, queries the Modrinth API for a newer version before each launch.
    - If an update is available, re-runs the mod download and config extraction steps.
    - Then starts the Forge server.

### `server/run.sh` — Generated launch script

- Reads `.setup-config.json` for auto-update setting and current version.
- Checks the Modrinth API for the latest release of the pack slug.
- If a newer version exists and auto-update is enabled, downloads and applies the update.
- Launches the server with the JVM arguments from `user_jvm_args.txt`.

## Output layout

```
server/
  config/                  ← mod config overrides from the mrpack
  libraries/               ← Forge runtime libraries (generated by installer)
  mods/
    *.jar                  ← server-side mod JARs
    .connector/            ← Connector remapping cache (generated at first boot)
    disabled_clientonly/   ← client-only mods moved here to prevent crashes
  world/                   ← generated on first boot
  eula.txt                 ← written if --agree-eula was passed
  run.sh                   ← generated launch + auto-update script
  user_jvm_args.txt        ← JVM memory settings (set -Xms/-Xmx here)
  server.properties        ← generated on first boot
  .setup-config.json       ← setup metadata (slug, version, auto-update)
```

## Notes

- Only **Forge** is supported as a mod loader currently.
- The script requires `bash`, `curl`, `unzip`, `python3`, and `java` (17+) to be available. The `python3` dependency is used solely for JSON parsing and is a candidate to be replaced with `jq` in a future update.
- Auto-update only applies to the mod files and config overrides — it does not reinstall Forge unless the Minecraft/Forge version changes.
- The EULA flag (`--agree-eula`) exists for automation convenience. You are still personally responsible for compliance with the [Minecraft EULA](https://aka.ms/MinecraftEULA).
